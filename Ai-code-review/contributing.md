# Code Style Guide

- All Python functions must have a docstring that explains their purpose, arguments, and return value.
- Use 4 spaces for indentation.
- Variable names should be in snake_case.
- Imports should be sorted alphabetically.
- All new features must be accompanied by at least one test case.

---

# Pull Request Best Practices

- The PR title should be concise and descriptive.
- The PR body should explain the problem being solved and how the code fixes it.
- A link to the associated Jira ticket or issue should be included.
